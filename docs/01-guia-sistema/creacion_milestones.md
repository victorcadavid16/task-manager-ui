# Gestión y Cronograma de Milestones (Hitos)

¡Hola, equipo de desarrollo! Si las **Issues** son los ladrillos de nuestra construcción, el **Milestone (Hito)** es el plano que nos dice cuándo terminar una habitación completa. Sin hitos, solo estamos haciendo tareas sueltas; con hitos, estamos alcanzando metas reales.

---

## 1. ¿Qué es un Milestone y para qué sirve?

Es un contenedor de tareas que tiene una fecha de entrega y un objetivo específico. Sirve para agrupar todas las **Issues** necesarias para completar una fase del proyecto.

- **Importancia:** Nos da un porcentaje real de progreso. No podemos decir *"ya casi acabamos"* si la barra del Milestone no está cerca del 100%.

---

## 2. Cómo crear un Milestone en GitHub (Paso a Paso)

El Líder (Arquitecto) es el encargado de configurar la ruta del equipo siguiendo estos pasos:

1. Ve a la pestaña **Issues** de tu repositorio  
2. Haz clic en el botón **Milestones** (arriba, junto a las etiquetas)  
3. Haz clic en el botón verde **New milestone**  
4. Llena los campos clave:

   - **Title:** Debe ser corto y profesional  
     *(Ej: v1.0 - Interfaz y Diseño)*  

   - **Due Date (Fecha de vencimiento):**  
     Fecha límite de entrega según el cronograma  

   - **Description:** Define qué se habrá logrado al finalizar  
     *(Ej: "Al finalizar este hito, la aplicación será responsiva y tendrá la estructura básica funcionando")*

---

## 3. Cómo se administra y se valida

- **Vinculación:**  
  Al crear o editar una **Issue**, el equipo debe seleccionar el Milestone correspondiente en el panel derecho de GitHub.  
  > Tarea que no tiene Milestone, es una tarea que no suma al progreso del equipo.

- **Validación Automática:**  
  El Milestone se valida solo. A medida que los desarrolladores terminan sus tareas y el Líder aprueba los **Pull Requests (PR)**, la barra de progreso subirá automáticamente.

---

## 4. ¿Cuándo está Abierto y cuándo Finalizado?

- **Abierto (Open):**  
  Estado de trabajo. Significa que aún hay tareas pendientes.

- **Finalizado (Closed):**  
  Un Milestone se cierra únicamente cuando la barra de progreso llega al 100%.  
  Esto indica que todo fue terminado, revisado por el Líder y aprobado.

---

## 5. ¿Qué ganamos usando Milestones?

- **Orden Profesional:** Trabajas por metas, no por intuición  
- **Calificación Transparente:** Si el Milestone está al 100%, tu resultado está asegurado  
- **Detección de Riesgos:** Permite reaccionar a tiempo si el progreso es bajo  
- **Perfil de Clase Mundial:** Demuestras capacidad de gestión del tiempo y entregas

## 6. Cronograma Sugerido: Proyecto

Para este proyecto, dividiremos el trabajo en 4 hitos clave. Úsalos como guía para crear tus Milestones en GitHub:

| Hito (Milestone)         | Objetivo Principal                     | ¿Qué tareas incluye?                                      |
|--------------------------|--------------------------------------|-----------------------------------------------------------|
| M1: Estructura y UI       | Base visual lista y responsiva        | Maquetación HTML, estilos CSS, diseño Mobile-First        |
| M2: Lógica y CRUD         | Que la app tenga funcionalidad        | Crear tarea, listar, borrar y validar campos              |
| M3: Persistencia / API    | Guardar datos reales                  | Uso de LocalStorage o consumo de API (fetch / axios)      |
| M4: Calidad y Deploy      | Producto final listo y en línea       | Limpieza de código, README final y despliegue en Vercel   |
